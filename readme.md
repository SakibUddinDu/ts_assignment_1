
১) What is the use of “enum”in TypeScript? Provide an example of a numeric and string enum.

TypeScript শিখতে গিয়ে কি enum শব্দটি তোমার চোখে পড়েছে? হয়তো তোমার মনে হয়েছে, এটা আবার কী কঠিন জিনিস! আরে বাবা, একদম চিন্তা কোরো না, এটা খুবই সহজ এবং কোডকে আরও গোছানো ও পড়তে সহজ করার জন্য একটা দারুণ টুল।
এই ব্লগে, আমরা একটা উদাহরণ দিয়ে দেখব enum আসলে কী, কেন আমরা এটা ব্যবহার করব, এবং এর নিউমেরিক (Numeric) ও স্ট্রিং (String) উদাহরণগুলো কী রকম হয়।
একটা বাস্তব উদাহরণ: ট্র্যাফিক লাইট!
ধরো, তুমি এমন একটি সফটওয়্যার বানাচ্ছো যা একটি ট্র্যাফিক লাইট (Traffic Light) সিস্টেমকে নিয়ন্ত্রণ করবে। একটি ট্র্যাফিক লাইটের কটি অবস্থা হতে পারে? তিনটি— লাল, হলুদ, এবং সবুজ।
কম্পিউটারের কাছে, এই তিনটি অবস্থাকে তুমি সংখ্যা দিয়েও বোঝাতে পারো:
লাল = 0
হলুদ = 1
সবুজ = 2
এখন, কোডের মধ্যে তুমি যদি লেখো if (lightState === 0) তাহলে কী হবে? কোডটি কাজ করবে ঠিকই, কিন্তু 0 মানে যে 'লাল', সেটা বোঝার জন্য তোমাকে বারবার মনে রাখতে হবে। যদি অন্য কোনো ডেভেলপার তোমার কোড দেখে, তাহলে তার জন্য এটা বোঝা খুবই কঠিন হবে।
এখান থেকেই enum এর প্রয়োজন শুরু হয়!

TypeScript-এ Enum কী?
Enum এর পুরো নাম হলো Enumeration। এটি TypeScript-এ এমন একটি ফিচার, যা তোমাকে নির্দিষ্ট কিছু নামযুক্ত ধ্রুবক (Named Constants) এর একটি সেট তৈরি করার সুযোগ দেয়।
সহজ কথায়, এটি তোমার কোডের কঠিন সংখ্যা বা স্ট্রিংগুলোকে সহজে মনে রাখার মতো নামে পরিবর্তন করে দেয়।
Enum ব্যবহার করার সুবিধা:
কোড পঠনযোগ্যতা বৃদ্ধি (Readability): 0 এর বদলে Light.Red লিখলে সাথে সাথেই বোঝা যায় তুমি কী বোঝাতে চাইছ।
ভুল হওয়ার সম্ভাবনা হ্রাস (Less Error-Prone): তুমি ভুল করে এমন কোনো সংখ্যা বা স্ট্রিং ব্যবহার করতে পারবে না যা ঐ সেটের অংশ নয়।
টাইপ-সেফটি (Type Safety): TypeScript নিশ্চিত করে যে তুমি একটি ভ্যারিয়েবলে enum এর বাইরে অন্য কোনো মান ব্যবহার করছ না।

১. নিউমেরিক Enum (Numeric Enum Example)
এটা enum এর সবচেয়ে সাধারণ প্রকার। এখানে, তুমি শুধু নামগুলো সংজ্ঞায়িত করবে এবং TypeScript স্বয়ংক্রিয়ভাবে প্রথম মানটিকে 0 দিয়ে শুরু করে ক্রমানুসারে সংখ্যা বসাবে।
উদাহরণ: ট্র্যাফিক লাইট
enum Light {
  Red,    // এর মান স্বয়ংক্রিয়ভাবে 0
  Yellow, // এর মান 1
  Green   // এর মান 2
}

let currentLight: Light = Light.Red; // currentLight এর মান হবে 0
console.log(currentLight); // আউটপুট: 0
console.log(Light.Green);  // আউটপুট: 2

 নোট : তুমি চাইলে যেকোনো নাম থেকে মান দেওয়া শুরু করতে পারো। যেমন: Red = 5, তাহলে Yellow হবে 6 এবং Green হবে 7।

২. স্ট্রিং Enum (String Enum Example)
অনেক সময়, আমরা সংখ্যার পরিবর্তে স্ট্রিং মান ব্যবহার করতে চাই। উদাহরণস্বরূপ, যখন আমরা ডাটাবেস বা সার্ভারের কাছ থেকে পাওয়া কোনো স্ট্যাটাস কোড নিয়ে কাজ করি।
স্ট্রিং enum এ প্রতিটি ধ্রুবকের জন্য তোমাকে অবশ্যই একটি স্ট্রিং মান (String Value) নির্ধারণ করতে হবে।
উদাহরণ: API স্ট্যাটাস কোড
ধরা যাক, তোমার সার্ভার বিভিন্ন অপারেশন শেষে এই স্ট্যাটাসগুলো পাঠায়:
TypeScript
enum ApiStatus {
  SUCCESS = "Operation_OK",
  FAILED = "Operation_FAIL",
  PENDING = "Operation_WAITING",
}

function checkStatus(status: ApiStatus): void {
  if (status === ApiStatus.SUCCESS) {
    console.log("কাজটি সফলভাবে সম্পন্ন হয়েছে!");
  } else {
    console.log("কাজটি সম্পন্ন হয়নি, বা অপেক্ষায় আছে।");
  }
}

checkStatus(ApiStatus.SUCCESS);


// আউটপুট: কাজটি সফলভাবে সম্পন্ন হয়েছে!

এখানে, তুমি কোডের মধ্যে ApiStatus.SUCCESS লিখছ, যা অনেক বেশি বোধগম্য। কিন্তু কোড চলার সময় এর মান হিসাবে ব্যবহৃত হচ্ছে "Operation_OK"।


২) Provide an example of using union and intersection types in TypeScript.

Union Types (|) - অথবা (OR)
Union Type ব্যবহার করে আমরা একটি ভ্যালুকে একাধিক সম্ভাব্য টাইপের যেকোনো একটি হওয়ার অনুমতি দিই। এটিকে একটি উল্লম্ব বার (|) ব্যবহার করে সংজ্ঞায়িত করা হয়।
উদাহরণ : ক্যাটারিং সার্ভিসের মেনু কার্ড
ধরুন, আপনি একটি ক্যাটারিং সার্ভিস চালাচ্ছেন। ক্লায়েন্টরা ডিনারের জন্য ভাত অথবা রুটি (কিন্তু একবারে যেকোনো একটি) অর্ডার করতে পারে।
ভাত একটি স্ট্রিং (string) টাইপের খাবার।
রুটি একটি স্ট্রিং (string) টাইপের খাবার।
ক্লায়েন্টের অর্ডার টাইপটি হবে: ভাত OR রুটি।
TypeScript উদাহরণ
TypeScript
// টাইপ সংজ্ঞায়িত করা
type MealOption = "Rice" | "Roti";

// ফাংশন যেখানে Union Type ব্যবহার হচ্ছে
function prepareMeal(order: MealOption): void {
  if (order === "Rice") {
    console.log("ভাত রান্না করা হচ্ছে।");
  } else if (order === "Roti") {
    console.log("রুটি তৈরি করা হচ্ছে।");
  } else {
    // এই ক্ষেত্রে TypeScript স্বয়ংক্রিয়ভাবে এই পরিস্থিতি এড়িয়ে যাবে
    // কারণ 'order' শুধুমাত্র "Rice" বা "Roti" হতে পারে।
    console.log("অর্ডারটি মেনুর বাইরে।");
  }
}

prepareMeal("Rice"); // আউটপুট: ভাত রান্না করা হচ্ছে।
// prepareMeal("Pulao"); // Error: 'Pulao' is not assignable to type 'MealOption'

মূল কথা: Union Type মানে হলো: এই ভ্যারিয়েবলটি A অথবা B অথবা C হতে পারে।
২. Intersection Types (&) - এবং (AND)
Intersection Type হলো একাধিক টাইপকে একত্রিত করে একটি নতুন টাইপ তৈরি করা। এই নতুন টাইপে একত্রিত সমস্ত টাইপেরই বৈশিষ্ট্য থাকতে হবে। এটি অ্যামপারস্যান্ড (&) ব্যবহার করে সংজ্ঞায়িত করা হয়।
উপমা: অ্যাডমিন প্যানেলের চাবি
ধরুন, একটি অফিসের অ্যাডমিন প্যানেলে প্রবেশ করতে হলে তোমার দুটি ভিন্ন কার্ডের প্রয়োজন:
ব্যবহারকারীর আইডি কার্ড (User ID Card)
সিকিউরিটি কি-কার্ড (Security Key Card)
তোমার অ্যাডমিন হিসেবে পরিচিত হওয়ার জন্য ব্যবহারকারীর আইডি কার্ড AND সিকিউরিটি কি-কার্ড দুটোই থাকতে হবে।
TypeScript উদাহরণ
প্রথমে আমরা দুটি ইন্টারফেস (কার্ড) তৈরি করি:
TypeScript
// আইডি কার্ডের বৈশিষ্ট্য
interface HasIDCard {
  employeeId: number;
  name: string;
}

// কি-কার্ডের বৈশিষ্ট্য
interface HasKeyCard {
  accessLevel: 'L1' | 'L2' | 'L3';
  keySerial: string;
}

// অ্যাডমিনের টাইপ: তাকে HasIDCard এবং HasKeyCard দুটোই হতে হবে
type Admin = HasIDCard & HasKeyCard;

const newAdmin: Admin = {
  employeeId: 101, // HasIDCard এর বৈশিষ্ট্য
  name: "Rahim",   // HasIDCard এর বৈশিষ্ট্য
  
  accessLevel: 'L3', // HasKeyCard এর বৈশিষ্ট্য
  keySerial: "XYZ-789" // HasKeyCard এর বৈশিষ্ট্য
};

// const incompleteAdmin: Admin = { name: "Karim", employeeId: 102 }; 
// Error: 'HasKeyCard' এর বৈশিষ্ট্যগুলো নেই

মূল কথা: Intersection Type মানে হলো: এই ভ্যারিয়েবলটির মধ্যে A এবং B এবং C এর সমস্ত বৈশিষ্ট্যই থাকতে হবে।
